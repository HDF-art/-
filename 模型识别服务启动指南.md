# 模型识别服务启动和测试指南

## 概述

本文档说明如何启动完整的图像识别服务链路,包括:
1. Python模型服务 (Flask)
2. Java后端服务 (Spring Boot)
3. 前端应用 (Vue)

## 架构说明

```
前端 (Vue) → 后端 (Java/Spring Boot) → 模型服务 (Python/Flask)
    ↓              ↓                        ↓
 用户界面      业务逻辑/数据库           SepResNet模型推理
```

## 六、问题排查

### 6.1 模型服务无法启动

**问题**: 缺少依赖或CUDA错误

**解决方案**:
```bash
pip install torch torchvision flask pillow
```

如果CUDA不可用,模型会自动使用CPU

### 6.2 后端无法连接模型服务

**问题**: Connection refused

**检查**:
1. 模型服务是否启动: `http://localhost:5000/health`
2. 端口是否被占用
3. 防火墙设置

**解决方案**:
- 确保模型服务先启动
- 检查 `application.yml` 中的URL配置

### 6.3 前端调用失败

**问题**: 401/403/500错误

**检查**:
1. 是否已登录
2. Token是否有效
3. 后端服务是否正常
4. 浏览器控制台错误信息

### 6.4 识别结果异常

**问题**: 返回"模拟数据"或置信度异常

**可能原因**:
1. 模型权重未正确加载
2. 图像预处理问题
3. 模型服务版本不匹配

**解决方案**:
- 检查模型服务日志
- 验证 `global_model.pth` 文件存在
- 测试简单的随机输入

## 十、注意事项

⚠️ **重要提醒**:

1. **模型权重**: 确保 `models/global_model.pth` 存在且正确
2. **mode参数**: SepResNet推理必须传递mode参数
3. **服务顺序**: 先启动模型服务,再启动后端
4. **端口占用**: 确保5000(模型)和8080(后端)端口可用
5. **图片格式**: 支持JPG/PNG,建议不超过5MB
